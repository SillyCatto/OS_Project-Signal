# Setup and Execution Guide

This document explains how to set up, run, and validate the Signal project on mCertiKOS.

## 1. Required Software, Tools, and Dependencies

Use a Linux environment (native Ubuntu or WSL2 Ubuntu) with the following packages.

### 1.1 Build and debug tools

- build-essential
- gcc-multilib
- gdb
- bear
- make
- python3

### 1.2 Virtualization and runtime tools

- qemu-kvm
- virt-manager
- virtinst
- libvirt-clients
- bridge-utils
- libvirt-daemon-system

### 1.3 Install commands

```bash
sudo apt update
sudo apt install -y build-essential gcc-multilib gdb bear
sudo apt install -y qemu-kvm virt-manager virtinst libvirt-clients bridge-utils libvirt-daemon-system
```

Enable virtualization service:

```bash
sudo systemctl enable --now libvirtd
```

Add your user to required groups:

```bash
sudo usermod -aG libvirt $USER
sudo usermod -aG kvm $USER
```

Important: log out and log in again after group updates.

## 2. Step-by-Step Project Setup

### 2.1 Get into project root

```bash
cd /path/to/OS_Project-Signal
```

Confirm key files exist:

```bash
ls
```

You should see entries like:

- Makefile
- make_image.py
- kern/
- user/
- boot/

### 2.2 Clean previous build artifacts (recommended)

```bash
make clean
```

### 2.3 Build project

```bash
make
```

If build succeeds, kernel/user images are generated by the project make pipeline.

## 3. How to Run the Project on a Local Machine

From the repository root:

### 3.1 Run in QEMU GUI mode

```bash
make qemu
```

### 3.2 Run in QEMU terminal-only mode

```bash
make qemu-nox
```

After boot, wait for the mCertiKOS shell prompt.

## 7. Troubleshooting

### Build fails due to missing packages

Run install commands again and ensure gcc-multilib is installed.

### QEMU launch issues

- Verify qemu-kvm is installed.
- Verify your user is in kvm and libvirt groups.
- Re-login after usermod group commands.

### Permission or virtualization issues

Check service status:

```bash
systemctl status libvirtd
```

If inactive, start it:

```bash
sudo systemctl start libvirtd
```

### Shell commands exist but tests do not run as expected

Rebuild from clean state:

```bash
make clean
make
```

Then rerun test cases in order.
